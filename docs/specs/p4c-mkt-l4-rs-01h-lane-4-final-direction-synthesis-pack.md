## 3. Final Direction Recommendation

### 3.1 Primary direction

`Guided Ledger / Route Spine`

- product truth fit: looks like workflow support, not authority theatre
- trust posture fit: calm-procedural baseline with clear escalation gravity
- readability fit: high contrast, low ornament, explicit panel hierarchy
- distinctiveness fit: ownable app-tile silhouette without house/key/people cliches
- shell + matter fit: supports both routine shell operations and higher-stakes matter escalation

### 3.2 Fallback direction

`Civic Folder / Cut Block Monogram`

- reserved for cases where users read the primary as too severe
- keeps semantic and hierarchy posture intact while softening visual temperature
- retains acceptable small-size icon performance, but trails primary in mixed-crowded launcher find speed

### 3.3 Explicit rejects

Rejected for final recommendation:

- handshake/people icon as default answer
- house/roof/key/door proptech motifs
- seal/crest/courthouse/gavel/badge authority motifs
- `Tabbed Path` as primary icon (fails mixed-crowded `24px` gate in RS-01G QA)
- celebratory fintech/hype treatment (neon, confetti-success, momentum theatrics)
- bland generic-admin flattening that removes matter/escalation seriousness

## 4. Final LandlordBuddy Wordmark Treatment

Fixed by project override: `LandlordBuddy`.

Primary treatment:

- `Public Sans 600`
- single-color rendering on consequential surfaces
- slight positive tracking to preserve `dB` seam readability at small sizes
- no split-color, no decorative effects, no pseudo-official styling

Fallback treatment:

- `Inter 600` with the same no-effects and contrast policy

Compact lockup behavior:

- `Full`: icon + `LandlordBuddy`
- `Compact`: icon + tightened `LandlordBuddy` spacing
- `Ultra-compact`: icon-only where readable text is no longer viable

## 6. Final Semantic Colour Direction

Direction: role-driven semantic palette with accessibility-first use constraints.

Final role set:

- `Neutral / Structure`: `Ink 900 #10262E`, `Slate 700 #2F485D`, `Surface 000 #FFFFFF`, `Surface 050 #F3F7F7`
- `Informational / Trust`: `Trust Blue #3F6A8A` (primary trust text/icon), `Calm Teal #6F9FB5` (secondary tint/fill)
- `Review / Warning`: `Warning Amber #C28C32`
- `Referral / Stop`: `Referral Rust #9D4227`

Hard usage controls:

- do not rely on color alone for state meaning
- do not use amber/teal as white-text body carriers where contrast drops below safe thresholds
- keep warning/review/referral meanings structurally distinct (panel weight, icon, label, placement)
- avoid success-green certainty language on consequential surfaces

## 7. Final Typography Direction

Primary typography system:

- headings/labels/state anchors: `Public Sans`
- body/supporting/help text: `Source Sans 3`

Fallback typography system:

## 8. Visual Doctrine Summary (Final)

1. Visual system must feel like workflow-support, not legal-authority theatre.
2. Shell surfaces should read routine, useful, and continuity-oriented.
3. Matter/escalation surfaces should read serious, bounded, and consequence-aware.
4. Trust cues are functional controls, not decorative chrome.
5. Readability under stress outranks novelty or hero-size styling.
6. No surface may imply filing power, official equivalence, or legal certification.
7. Branding must not outrank warning, boundary, referral, or handoff semantics.
8. Icon/wordmark decisions are valid only if launcher and small-size performance hold.

## 9. Shell-Mode vs Matter-Mode Behaviour

| Mode | Visual posture | Layout and contrast behaviour | Must-not-imply guardrail |
| --- | --- | --- | --- |
| `Shell mode` | calm, routine, operational continuity | lighter surfaces, stable structure, restrained accents, clear record/reminder affordances | generic PM-suite ambition, compliance clearance, official authority |
| `Matter / escalation mode` | serious, consequence-aware, bounded | heavier state panels, tighter hierarchy, explicit warning/referral/handoff separation, stronger contrast discipline | filing power, legal certification, triumphant finality |

## 10. Sample Surface Applications (Final)

1. Homepage / Entry (`L1-SCR-01` style)
- Shell-mode posture, clear bounded promise, no faux-authority cues.

2. Shell home / tenancy context (`SH-SCR-01/02` class)
- Routine continuity view for records, reminders, and active matters without PM-suite overreach.

3. Threshold check / notice preparation (`L1-SCR-05/06` class)
- Transition surface: routine context yields to higher-stakes matter framing with explicit state blocks.

4. Output review / official handoff (`L1-SCR-09` class)
- Matter-mode posture with external-owner clarity; `Prepared for handoff` remains anti-finality.

5. Warning / referral / stop surface (`L1-OVR-04` class)
- Highest-severity expression; ordinary handoff visual primacy is suppressed when referral/wrong-channel is active.

## 11. Implementation-Facing Visual Rules

`VR-01` Use semantic/renderer state labels directly; do not create generic "all good" visual collapse.

`VR-02` Keep trust cue and boundary cue physically adjacent to consequential action blocks.

`VR-03` Treat icon style as structural geometry only; no narrative illustration in launcher tile.

`VR-04` Keep wordmark effects off; contrast and spacing carry legibility.

`VR-05` Apply shell-mode and matter-mode panel weights distinctly; do not flatten them into one mood.

`VR-06` Preserve explicit visual distinction across blocked/review/referral/stale/live/wrong-channel families.

`VR-07` Use amber/rust only for semantic warning/referral roles, never as decorative atmosphere.

`VR-08` Maintain readable minimum typography and avoid condensed/lightweight cuts on consequential surfaces.

`VR-09` Suppress ordinary handoff prominence when referral-first or wrong-channel controls are active.

`VR-10` Keep export/reminder visuals continuity-oriented; never style them like official certification.

`VR-11` Enforce launcher QA gate: if `24px` mixed-crowded recognition regresses, icon change is not accepted.

`VR-12` Preserve anti-fake-certainty language posture in visual emphasis, not only copy.
