# ADR-L2-first-wave-copy-baseline

## Status
Accepted

## Decision
Lane 2 is closed for authoring and moves to implementation fidelity only.
The first-wave build copy baseline is the approved Lane 2 marketing handoff pack.
Build must preserve protected lines, state-family distinctions, CTA hierarchy, trust-cue binding, and warning-family consequences.

## Why
Lane 2 reached an acceptable semantic freeze.
Further general authoring would increase drift risk without meaningful product value.
The project now benefits more from implementation fidelity than further prose iteration.

## What is frozen
- protected lines
- state labels and state distinctions
- CTA hierarchy
- trust-cue binding
- warning-family consequences
- stale/live-confirmation distinction
- scoped-hold posture where relevant to first-wave copy

## What is not frozen
- narrow optimization/testing questions that do not change protected lines or semantic controls
- screen-specific polish that stays fully inside the frozen semantic fence

## Reopen triggers
Reopen Lane 2 only if:
- a protected line must change
- a state label, CTA tier, boundary line, trust cue, or warning consequence must change
- a trust-cue-bound state must appear without its required cue
- a new screen cannot be extended cleanly from the existing pattern library
- build attempts to strengthen legal certainty, filing/submission, official equivalence, or privacy/security precision claims

## Consequence
Lane 2 now moves into implementation fidelity only.
