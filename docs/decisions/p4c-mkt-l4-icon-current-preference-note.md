# P4C Lane 4 Icon Current Preference Note

Date: 2026-05-04
Status: current working preference correction

## Decision / correction

The current working preference for the LandlordBuddy app icon/logo direction is the `Document + House` / `House + Document` compound mark.

## What this changes

Earlier repo Lane 4 artifacts from `P4C-MKT-L4-RS-01G` and `P4C-MKT-L4-RS-01H` record a limited launcher-first Route Spine pass and describe Route Spine as primary inside that pass.

Those Route Spine recommendations should now be treated as historical test evidence, not the current working icon preference.

## Current interpretation

- `Document + House` / `House + Document` is the current working preference.
- Route Spine may remain a control, fallback exploration, or UI motif candidate.
- Route Spine should not be treated as the current primary app tile/logo direction unless a later accepted decision explicitly reopens and reverses this preference.

## Why

The House/Doc direction better preserves the needed category signal: rental property / tenancy plus documents, records, evidence, notices, and workflow support.

## Guardrails preserved

The current House/Doc direction must still avoid:

- house-only real estate cliche
- key/lock/property-agent styling
- seal, courthouse, gavel, badge, or fake-official styling
- certification/tickmark implication
- clutter that fails at small launcher sizes

## Validation requirement

Before visual freeze, House/Doc variants still need the same practical proof expected of any icon direction:

- 16px / 20px / 24px / 32px / 48px / 64px checks
- light, dark, and crowded launcher checks
- category recognition test
- misclassification check against real estate, document-signing, government, legal, and generic workflow apps

## Revisit trigger

Revisit only if House/Doc variants fail small-size recognition, category recognition, or anti-fake-official tests, or if a later accepted Lane 4 synthesis explicitly supersedes this correction.
